83  * info_hash : urlencoded 20-byte SHA1 hash of the value of the info key from the Metainfo file
84  * peer_id: urlencoded 20-byte string used as a unique ID for the client, generated by the client at startup.
85  * port: The port number that the client is listening on.
86  * uploaded: The total amount uploaded in base ten ASCII.
87  * downloaded: The total amount downloaded in base ten ASCII
88  * left: The number of bytes this client still has to download in base ten ASCII.
89  * compact: Setting this to 1 indicates that the client accepts a compact response.
90  * no_peer_id: Indicates that the tracker can omit peer id field in peers dictionary.
91  * event: If specified, must be one of started, completed, stopped
92  * started: The first request to the tracker must include the event key with this value.
93  * stopped: Must be sent to the tracker if the client is shutting down gracefully.
